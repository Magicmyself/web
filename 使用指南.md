# MiKun Vision 摄影工作室网站使用指南

## 📁 项目结构

```
webtext/
├── assets/           # 原始资源文件夹
│   ├── config/       # 配置文件
│   ├── images/       # 图片文件
│   └── 配置文件使用指南.md
├── dist/             # 构建后的文件（部署用）
├── public/           # 静态资源文件
├── src/              # 源代码
├── build.bat         # 构建脚本
├── start.bat         # 启动服务器脚本
├── 启动服务器.bat     # 启动服务器脚本（中文）
├── stop.bat          # 停止服务器脚本
└── server.mjs        # 服务器配置
```

## 🎨 修改网站内容

### 修改文字内容

**1. 网站基本信息**
- 文件：`assets/config/site-info.json`
- 内容：网站标题、副标题、描述等

**2. 作品展示**
- 文件：`assets/config/works.json`
- 内容：作品图片、标题、描述等

**3. 价格套餐**
- 文件：`assets/config/packages.json`
- 内容：套餐名称、价格、包含内容等

**4. 团队成员**
- 文件：`assets/config/members.json`
- 内容：成员姓名、角色、头像、描述等

**5. 联系信息**
- 文件：`assets/config/contact.json`
- 内容：QQ、微信、邮箱等

**6. 规则说明**
- 文件：`assets/config/rules.json`
- 内容：约单须知、互勉规则等

**7. 轮播海报**
- 文件：`assets/config/posters.json`
- 内容：海报图片、标题、描述等

**8. 热门IP**
- 文件：`assets/config/popular-ips.json`
- 内容：热门IP列表

### 修改图片内容

**1. 作品图片**
- 文件夹：`assets/images/works/`
- 命名规则：1.jpg, 2.jpg, 3.jpg...
- 对应配置：`assets/config/works.json` 中的 `imageUrl` 字段

**2. 团队成员头像**
- 文件夹：`assets/images/members/`
- 命名规则：member1.jpg, member2.jpg...
- 对应配置：`assets/config/members.json` 中的 `avatarUrl` 字段

**3. 轮播海报**
- 文件夹：`assets/images/posters/`
- 命名规则：poster1.jpg, poster2.jpg...
- 对应配置：`assets/config/posters.json` 中的 `imageUrl` 字段

## 🚀 更新网站流程

### 本地更新流程

1. **修改内容**：
   - 修改 `assets/config/` 中的配置文件
   - 更新 `assets/images/` 中的图片文件

2. **重新构建**：
   - 双击运行 `build.bat` 脚本
   - 等待构建完成（会显示 "Build completed!"）

3. **启动服务器**：
   - 双击运行 `启动服务器.bat` 脚本
   - 访问 `http://localhost:3000` 查看效果

4. **停止服务器**：
   - 按任意键停止服务器
   - 或运行 `stop.bat` 脚本

### 线上部署流程

1. **本地构建**：
   - 按照上述步骤完成本地更新和构建

2. **上传文件**：
   - 将 `dist/` 文件夹中的所有文件上传到您的托管平台（如 Cloudflare Pages）

3. **清除缓存**：
   - 如果使用 Cloudflare，登录控制台清除缓存
   - 等待几分钟后访问网站查看效果

## 🔧 常见问题及解决方案

### 1. 图片不显示
- **原因**：图片路径错误或未重新构建
- **解决**：
  - 检查图片文件名是否与配置文件中的路径一致
  - 重新运行 `build.bat` 构建项目

### 2. 文字内容未更新
- **原因**：未重新构建或缓存问题
- **解决**：
  - 重新运行 `build.bat` 构建项目
  - 清除浏览器缓存后刷新页面

### 3. 服务器启动失败
- **原因**：端口被占用
- **解决**：
  - 运行 `stop.bat` 脚本停止占用端口的进程
  - 重新启动服务器

### 4. 构建脚本闪退
- **原因**：权限问题或路径错误
- **解决**：
  - 以管理员身份运行 `build.bat`
  - 检查 Node.js 路径是否正确

### 5. 图片更新后网站未显示
- **原因**：缓存问题
- **解决**：
  - 重新构建项目
  - 清除 Cloudflare 缓存
  - 强制刷新浏览器（Ctrl+Shift+R）

## 💡 最佳实践

1. **图片优化**：
   - 压缩图片大小，建议使用 WebP 格式
   - 保持图片分辨率适中，避免过大的文件

2. **配置文件管理**：
   - 定期备份配置文件
   - 修改前先复制一份原始文件

3. **版本管理**：
   - 每次重大更新后记录修改内容
   - 参考 `CHANGELOG.md` 文件

4. **性能优化**：
   - 启用懒加载功能（已实现）
   - 合理使用图片尺寸
