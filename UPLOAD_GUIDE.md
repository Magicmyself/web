# 网站更新与部署指南

本指南将详细介绍如何更新和重新部署MiKun Vision摄影工作室网站的完整流程。

## 目录结构

```
webtext/
├── assets/           # 资源文件
│   ├── config/       # 配置文件（JSON格式）
│   │   ├── members.json     # 团队成员配置
│   │   ├── packages.json    # 套餐配置
│   │   ├── works.json       # 作品配置
│   │   ├── site-info.json   # 网站基本信息
│   │   ├── contact.json     # 联系信息
│   │   └── ...
│   └── images/       # 图片文件
│       ├── members/  # 团队成员相关图片
│       │   ├── member1.jpg   # 成员头像
│       │   ├── member2.jpg   # 成员头像
│       │   └── payment/      # 收款码图片
│       │       ├── magic_wechat.jpg   # Magicmyself微信收款码
│       │       ├── magic_alipay.jpg   # Magicmyself支付宝收款码
│       │       └── ...
│       └── works/    # 作品展示图片
├── functions/        # 后端API函数
│   └── api/          # API路由目录
│       └── send-email.js  # 邮件发送接口
├── public/           # 静态资源（自动生成）
├── src/              # 前端源代码
├── build.bat         # 构建脚本
├── start.bat         # 启动服务器脚本
├── deploy.bat        # 部署脚本（推荐）
└── UPLOAD_GUIDE.md   # 本指南
```

## 1. 更新网站内容

### 1.1 更新团队成员信息
1. **修改成员配置**：
   - 打开 `assets/config/members.json` 文件
   - 修改成员姓名、角色、邮箱等信息
   - 确保 `payment` 字段中的收款码路径正确

2. **更新成员头像**：
   - 将新头像图片复制到 `assets/images/members/` 目录
   - 文件名格式：`member1.jpg`, `member2.jpg` 等
   - 建议尺寸：200x200像素

3. **更新收款码**：
   - 将新收款码图片复制到 `assets/images/members/payment/` 目录
   - 文件名格式：`{成员姓名小写}_wechat.jpg` 和 `{成员姓名小写}_alipay.jpg`
   - 示例：`magic_wechat.jpg`, `magic_alipay.jpg`
   - 建议尺寸：300x300像素

### 1.2 更新作品展示
1. **修改作品配置**：
   - 打开 `assets/config/works.json` 文件
   - 添加或修改作品的标题、分类、描述等

2. **上传作品图片**：
   - 将新作品图片复制到 `assets/images/works/` 目录
   - 文件名格式：`1.jpg`, `2.jpg` 等
   - 建议尺寸：1200x800像素
   - 支持格式：jpg, jpeg, png

### 1.3 更新套餐信息
- 打开 `assets/config/packages.json` 文件
- 修改套餐价格、内容等信息

### 1.4 更新网站基本信息
- 打开 `assets/config/site-info.json` 文件
- 修改网站名称、标语等信息

- 打开 `assets/config/posters.json` 文件
- 修改轮播图标题、副标题和图片URL

### 1.5 更新联系信息
- 打开 `assets/config/contact.json` 文件
- 修改电话、邮箱、地址等联系信息

## 2. 构建项目

### 2.1 运行构建脚本
1. **双击运行 `build.bat`**
2. **等待构建完成**
   - 脚本会自动：
     - 清除旧的静态资源
     - 复制新的图片文件到 `public` 目录
     - 构建生产版本
     - 显示构建完成信息

### 2.2 验证构建结果
- 构建成功后，会在 `dist` 目录生成优化后的静态文件
- 检查终端输出，确保没有错误信息

## 3. 部署到Cloudflare Pages

### 3.1 运行部署脚本
1. **双击运行 `deploy.bat`**
2. **等待部署完成**
   - 脚本会自动：
     - 检查Git状态
     - 添加所有更改
     - 提交代码
     - 推送到远程仓库
     - 显示部署完成信息

### 3.2 检查部署状态
1. **登录Cloudflare Dashboard**
2. **进入「Workers & Pages」**
3. **选择您的项目**
4. **查看「Deployments」标签页**
   - 确认部署状态为「成功」（绿色对勾）
   - 部署通常需要1-2分钟完成

## 4. 测试网站

1. **访问网站**：
   - 打开浏览器，访问您的网站域名
   - 例如：`https://mikunvision.site`

2. **测试功能**：
   - 检查所有页面是否正常加载
   - 测试预约流程：选择套餐 → 选择团队成员 → 查看收款码
   - 确认图片是否正确显示
   - 验证所有链接是否正常工作

3. **检查响应式设计**：
   - 在不同设备上测试网站（手机、平板、桌面）
   - 确保布局在各种屏幕尺寸下都正常

## 6. 常见问题与解决方案

### 6.1 图片不显示
- **原因**：图片路径错误或未同步
- **解决方案**：
  1. 确保图片文件已上传到正确的目录
  2. 检查配置文件中的图片路径是否正确
  3. 重新运行 `build.bat` 同步图片

### 6.2 构建失败
- **原因**：配置文件格式错误或依赖问题
- **解决方案**：
  1. 检查配置文件的JSON格式是否正确
  2. 确保所有依赖已安装（运行 `npm install`）
  3. 查看终端输出的错误信息

### 6.3 部署失败
- **原因**：Git仓库连接问题或代码错误
- **解决方案**：
  1. 检查Cloudflare Pages的部署状态和错误信息
  2. 确保Git仓库连接正常
  3. 检查代码是否有语法错误

### 6.4 字符乱码
- **原因**：编码设置问题
- **解决方案**：
  1. 使用 `deploy.bat` 脚本（英文显示，避免编码问题）
  2. 确保配置文件使用UTF-8编码

## 7. 图片优化建议

- **图片格式**：建议使用JPEG格式，质量70-80%
- **图片尺寸**：
  - 头像：200x200像素
  - 收款码：300x300像素
  - 作品：1200x800像素
- **文件命名**：使用数字或简短的英文名称
- **压缩工具**：可以使用在线工具如TinyPNG进行压缩
- **文件大小**：建议单张图片不超过2MB

## 8. 注意事项

### 8.1 配置文件格式
- 确保JSON格式正确，使用在线JSON验证工具检查
- 保持配置文件的结构一致
- 不要删除必要的字段

### 8.2 安全注意事项
- 不要在配置文件中存储敏感信息
- 收款码图片建议设置为适当的大小，避免被滥用

### 8.3 故障排查
- 如果更新后网站显示异常，检查配置文件格式
- 如果图片不显示，检查图片路径和文件名是否正确
- 如果部署失败，查看Cloudflare Pages的构建日志
