# 网站配置与管理指南

## 目录结构说明

### 核心目录结构
```
webtext/
├── assets/           # 配置文件和原始图片（用于编辑）
│   ├── config/       # 配置文件（JSON格式）
│   └── images/       # 原始图片文件
│       ├── works/    # 作品图片
│       ├── members/  # 成员图片
│       └── posters/  # 轮播图海报
├── public/           # 静态资源（构建时会复制到dist）
│   └── assets/       # 静态资源目录
│       └── images/   # 静态图片（与assets/images结构相同）
│           ├── works/
│           ├── members/
│           └── posters/
├── dist/             # 构建输出目录（自动生成）
└── server.mjs        # 本地服务器脚本
```

## 如何管理内容

### 1. 修改网站文字内容

所有文字内容都保存在 `assets/config/` 目录下的 JSON 文件中：

| 配置文件 | 用途 |
|---------|------|
| site-info.json | 网站基本信息（名称、标语、主题色等） |
| posters.json | 轮播图海报配置 |
| popular-ips.json | 热门IP标签 |
| works.json | 作品展示配置 |
| packages.json | 价格套餐配置 |
| rules.json | 约单须知配置 |
| members.json | 团队成员配置 |
| contact.json | 联系信息配置 |

**修改步骤**：
1. 打开对应配置文件
2. 修改内容（保持JSON格式正确）
3. 保存文件
4. 重新构建项目

### 2. 添加和修改图片

**图片管理流程**：
1. **添加图片**：将图片文件复制到 `assets/images/` 对应子目录
2. **同步图片**：运行 `E:\node\npm run sync` 同步图片到public目录
3. **配置引用**：在配置文件中设置正确的图片路径
4. **重新构建**：运行构建命令使修改生效

**图片目录说明**：
- `assets/images/works/` - 作品图片
- `assets/images/members/` - 成员头像
- `assets/images/posters/` - 轮播图海报

**图片路径格式**：
```json
"imageUrl": "assets/images/works/1.jpg"
```

## 如何预览和构建网站

### 1. 启动本地服务器

```bash
# 启动服务器
E:\node\node server.mjs

# 访问地址
http://localhost:3000
```

### 2. 重新构建项目

当修改了配置文件或图片后，需要重新构建项目：

```bash
# 标准构建（无详细输出）
E:\node\npx vite build

# 详细构建（显示构建过程和结果）
E:\node\npm run build:verbose

# 构建完成后刷新浏览器即可看到更新
```

### 3. 停止服务器

在终端中按 `Ctrl+C` 停止服务器。

## 常见问题解决

### 图片不显示
- 检查图片文件是否存在于 `assets/images/` 对应目录
- 检查配置文件中的图片路径是否正确
- 检查 `public/assets/images/` 目录中是否有对应图片
- 重新运行构建命令

### 配置文件修改不生效
- 检查 JSON 格式是否正确（可使用在线 JSON 验证工具）
- 重新运行构建命令
- 刷新浏览器缓存

### 服务器无法启动
- 检查 Node.js 是否安装正确
- 检查端口 3000 是否已被占用
- 重新启动服务器

## 最佳实践

1. **备份配置文件**：修改前建议备份原始配置文件
2. **图片命名**：使用简单的数字或英文名称，避免中文和特殊字符
3. **图片格式**：推荐使用 JPG 或 PNG 格式
4. **图片大小**：建议优化图片大小，避免过大影响加载速度
5. **定期构建**：每次修改后都运行构建命令确保更改生效

## 快速修改指南

### 修改作品展示
1. 在 `assets/images/works/` 中添加图片
2. 在 `assets/config/works.json` 中添加或修改作品配置
3. 运行 `E:\node\npx vite build` 重新构建
4. 刷新浏览器查看更新

### 修改轮播图
1. 在 `assets/images/posters/` 中添加图片
2. 在 `assets/config/posters.json` 中添加或修改海报配置
3. 运行 `E:\node\npx vite build` 重新构建
4. 刷新浏览器查看更新

### 修改团队成员
1. 在 `assets/images/members/` 中添加头像图片
2. 在 `assets/config/members.json` 中添加或修改成员配置
3. 运行 `E:\node\npx vite build` 重新构建
4. 刷新浏览器查看更新

## 技术说明

- **构建工具**：Vite 6.2.0
- **前端框架**：React 19.0.0 + TypeScript
- **样式框架**：Tailwind CSS 4.1.14
- **本地服务器**：Express

网站使用静态构建方式，所有修改都需要通过重新构建才能生效。这确保了网站的性能和安全性，适合部署到任何静态网站托管服务。
